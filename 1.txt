Revision: c4d71ae4f9f05
###[GridNG] Build up NGGridSizingTree for subgrid layout
>>>>Detail:
In order to correctly perform subgrid layout we need to do a significant
refactor in NGGrid to allow the topmost grid container to size the whole
grid tree. This is required since we need to provide the contribution
size of every subgrid up to the same step in the parent grid's sizing
algorithm, which we are not able to easily do in `ComputeMinMaxSizes`
nor `Layout` because it expects grid sizing to be finalized.

This CL introduces `NGGridSizingTree`, a class representing a tree of
multiple nested grids, with the root of the tree being the topmost grid
and the rest of the nodes being subgrids (at least one subgridded axis).

Our tree structure is "flattened", meaning it's contained in a single
vector with each tree node storing a pointer to its parent (except the
position 0, which is the root) and built in such a way that the nodes of
every subtree are contiguous in the vector. To perform grid sizing, each
node stores data for their respective grid's algorithm, e.g., its layout
track collections, grid items, placement data, etc.

###Track PrintJob lifetimes in printing browser tests
>>>>Detail:
This is a speculative fix for flaky crashes related to print jobs still
being pending on test termination.

Extend the PrintJob::Observer class to include observation of when a
PrintJob is being destroyed.

Expose a CreatePrintJob() method from PrintViewManagerBase for creation
of PrintJob objects.  This allows tests to override this to include
their own observer of the PrintJob.  This is needed instead of relying
on the observer already used by PrintViewManagerBase because that class
removes itself from observation once the print job is released, which
is too soon for knowing when it is safe to terminate tests.

Update SystemAccessProcessPrintBrowserTestBase to become a PrintJob
observer and track PrintJob lifetimes.  This can be used to ensure that
all PrintJobs ever created have been deleted before test termination.

Ensuring PrintJob termination should be more reliable than checking for
PrintJobWorker class having been stopped, as there is still significant
cleanup work that has to happen after that event.

Change the tests to no longer include tracking of OnDidStop() as part
of criteria for when to exit the testing loop.
